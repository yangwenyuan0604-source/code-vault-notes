# 数据结构期末速成

> [!important] 考试重点分布
> 线性表、树、图、查找、排序五大块，占总分 80%+。时间复杂度分析贯穿全卷。

---

## 一、绪论

### 1.1 数据结构三要素

- **逻辑结构**：线性结构（表、栈、队列） vs 非线性结构（树、图）
- **存储结构**：顺序存储、链式存储、索引存储、散列存储
- **运算**：增删改查

### 1.2 算法复杂度

> [!tip] 必背公式
> - 一条语句执行次数 × 每条语句时间 = 总时间
> - 只取最高阶，忽略常数系数

| 常见复杂度 | 名称 | 典型算法 |
|:---|:---|:---|
| $O(1)$ | 常数阶 | 哈希查找 |
| $O(\log n)$ | 对数阶 | 二分查找 |
| $O(n)$ | 线性阶 | 顺序查找 |
| $O(n \log n)$ | 线性对数阶 | 快排、归并 |
| $O(n^2)$ | 平方阶 | 冒泡、插入 |
| $O(2^n)$ | 指数阶 | 递归求斐波那契 |

**复杂度排序**：$O(1) < O(\log n) < O(n) < O(n\log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$

## 二、线性表

### 2.1 顺序表

- 逻辑相邻 = 物理相邻
- **存取** $O(1)$，**插入/删除** $O(n)$（需移动元素）
- 按值查找 $O(n)$
- 空间连续，随机访问

### 2.2 单链表

```mermaid
graph LR
    head[头指针] --> A[数据|next] --> B[数据|next] --> NULL[NULL]
```

- **存取** $O(n)$，**头插/头删** $O(1)$
- 按值查找 $O(n)$
- 不要求连续空间，无随机访问

> [!warning] 头插法 vs 尾插法
> - 头插法：新结点始终插在头部 → 输出顺序与输入**相反**
> - 尾插法：需要尾指针 → 输出顺序与输入**相同**

### 2.3 双链表

```c
typedef struct DNode {
    int data;
    struct DNode *prior, *next;
} DNode;
```

- 可双向遍历
- 插入/删除需修改 prior 和 next 两个指针
- **删除结点** 已知结点指针时 $O(1)$

### 2.4 循环链表

- 单循环：尾结点 next 指向头结点
- 双循环：头 prior 指向尾，尾 next 指向头
- **判空**：`L->next == L`

### 2.5 顺序表 vs 链表

| | 顺序表 | 链表 |
|:---|:---|:---|
| 存取 | $O(1)$ ✅ | $O(n)$ ❌ |
| 插入/删除 | $O(n)$ ❌ | $O(1)$ ✅ |
| 空间 | 连续，可能浪费 | 分散，有指针开销 |
| 适用 | 查多改少 | 改多查少 |

## 三、栈和队列

### 3.1 栈 (Stack)

- **LIFO**（后进先出）
- 顺序栈：top 指针指向栈顶
- 链栈：单链表，头插头删

> [!note] 栈的核心操作
> - `push` 入栈（先判满）
> - `pop` 出栈（先判空）
> - 时间复杂度均为 $O(1)$

**出栈序列数（n 个元素）**：卡特兰数 $\frac{1}{n+1}C_{2n}^{n}$

### 3.2 队列 (Queue)

- **FIFO**（先进先出）
- 顺序队列：front 指向队头，rear 指向队尾

> [!warning] 循环队列必考
> - **入队**：`rear = (rear + 1) % maxSize`
> - **出队**：`front = (front + 1) % maxSize`
> - **队满**：`(rear + 1) % maxSize == front`（牺牲一个单元）
> - **队长**：`(rear - front + maxSize) % maxSize`
> - **队空**：`front == rear`

- 链队列：带首尾指针的单链表

### 3.3 栈的应用

- 括号匹配
- 表达式求值（中缀转后缀）
- 递归（函数调用栈）
- DFS（深度优先搜索）

### 3.4 队列的应用

- BFS（广度优先搜索）
- 缓冲区、排队系统
- 层序遍历二叉树

## 四、串

### 4.1 基本概念

- 串：零个或多个字符组成的有限序列
- 串的长度:串中字符的个数n 
- 字符在串中的位置:该字符在串中的序号
- 子串在主串中的位置以子串的第一个字符在主串中的位置来表示
- 
- 空串 `""` ≠ 空格串 `" "`
- 子串：串中任意连续字符的子序列

### 4.2 串的存储

- 定长顺序存储
- 堆分配存储（动态）
- 块链存储

### 4.3 模式匹配

#### 暴力匹配（BF）

- 最坏时间复杂度：$O(n \times m)$
- 每次失配，主串指针回溯

#### KMP 算法

> [!tip] KMP 核心
> **主串指针不回溯**，利用 next 数组调整模式串位置。

- 时间复杂度：$O(n + m)$
- **next 数组**：表示失配时模式串应该跳到哪个位置
- **nextval 数组**：next 的优化版

```c
// 求 next 数组（考试手算）
// next[j] = 模式串位置 1~j-1 的最长相等前后缀长度 + 1
// 规定 next[1] = 0
```

| 位置 j | 1 | 2 | 3 | 4 | 5 |
|:---|:---|:---|:---|:---|:---|
| 模式串 | a | b | a | b | a |
| next[j] | 0 | 1 | 1 | 2 | 3 |

## 五、数组与广义表

### 5.1 数组

- 多维数组 → 一维存储（行优先 / 列优先）
- **对称矩阵压缩**：只存下三角，$n(n+1)/2$ 个元素
- **三角矩阵**：存 $n(n+1)/2 + 1$（多存一个常量）
- **稀疏矩阵**：三元组表 / 十字链表

> [!note] 地址计算
> 行优先：$LOC(i,j) = LOC(0,0) + (i \times n + j) \times L$

### 5.2 广义表

- $LS = (a_1, a_2, ..., a_n)$
- **表头 Head**：第一个元素
- **表尾 Tail**：除去表头剩余部分组成的表
- 可以是递归定义 $L = (a, L)$
- 深度 = 括号嵌套最大层数

## 六、树与二叉树

### 6.1 基本概念

- 结点的**度**：子结点数
- 树的**度**：max(所有结点度)
- 叶结点：度为 0
- 树高：最大层次

> [!tip] 树的性质（必考）
> - 总结点数 = 所有结点度之和 + 1
> - 度为 m 的树，第 i 层最多 $m^{i-1}$ 个结点

### 6.2 二叉树

**性质**：

> [!important] 二叉树 5 大性质
> 1. 第 i 层最多 $2^{i-1}$ 个结点
> 2. 深度为 k 最多 $2^k - 1$ 个结点
> 3. $n_0 = n_2 + 1$（叶结点数 = 度为 2 的结点数 + 1）
> 4. n 个结点的完全二叉树深度 $\lfloor \log_2 n \rfloor + 1$
> 5. 完全二叉树编号 i 的结点：左孩子 $2i$，右孩子 $2i+1$，父结点 $\lfloor i/2 \rfloor$

- 满二叉树：每层全满
- 完全二叉树：最后一层靠左连续

### 6.3 二叉树的存储

- **顺序存储**：按编号存数组，仅适合完全二叉树
- **链式存储**：lchild + data + rchild

### 6.4 二叉树遍历（必考大题）

| 遍历 | 顺序 | 口诀 |
|:---|:---|:---|
| 先序 | 根 → 左 → 右 | N L R |
| 中序 | 左 → 根 → 右 | L N R |
| 后序 | 左 → 右 → 根 | L R N |
| 层次 | 逐层从左到右 | BFS |

> [!warning] 遍历恢复二叉树
> - **先序 + 中序 → 二叉树** ✅ 唯一
> - **后序 + 中序 → 二叉树** ✅ 唯一
> - 先序 + 后序 → 二叉树 ❌ 不唯一

**非递归遍历**：用栈模拟递归过程

### 6.5 线索二叉树

- 利用空指针域存放前驱/后继
- ltag=0 → lchild 指向左孩子；ltag=1 → lchild 指向前驱
- rtag=0 → rchild 指向右孩子；rtag=1 → rchild 指向后继

### 6.6 树与森林

- **树 → 二叉树**：兄弟相连，断父连长子（左孩子右兄弟）
- **森林 → 二叉树**：各树先分别转二叉树，再根相连
- **树的遍历**：
  - 先根遍历 = 对应二叉树的先序遍历
  - 后根遍历 = 对应二叉树的中序遍历

### 6.7 哈夫曼树（必考大题）

- **WPL**（带权路径长度）最小
- 构造：每次选最小的两个权值合并
- 哈夫曼编码：左 0 右 1，前缀编码

> [!note] 哈夫曼树性质
> - n 个叶子结点的哈夫曼树共有 $2n-1$ 个结点
> - 没有度为 1 的结点
> - WPL = 所有非叶结点权值之和（考试速算技巧）

## 七、图

### 7.1 基本概念

- **完全图**：无向图 $\frac{n(n-1)}{2}$ 条边，有向图 $n(n-1)$ 条弧
- **连通**：无向图中任意两点可达
- **强连通**：有向图中任意两点互相可达
- **生成树**：包含所有顶点的极小连通子图（n-1 条边）

### 7.2 图的存储

| 存储方式 | 空间 | 适用场景 |
|:---|:---|:---|
| 邻接矩阵 | $O(n^2)$ | 稠密图 |
| 邻接表 | $O(n+e)$ | 稀疏图 |

> [!tip] 邻接表特点
> - 无向图每条边出现**两次**
> - 求顶点出度方便，入度需遍历

### 7.3 图的遍历

- **DFS**（深度优先）：递归/栈，类似树的先序遍历
- **BFS**（广度优先）：队列，类似树的层序遍历

> [!note] 时间复杂度
> - 邻接矩阵：$O(n^2)$（遍历矩阵）
> - 邻接表：$O(n+e)$（遍历邻接表）

### 7.4 最小生成树

#### Prim 算法

- 从顶点出发，每次选择与当前集合最近的点加入
- 时间复杂度 $O(n^2)$
- 适合**稠密图**

#### Kruskal 算法

- 从边出发，每次选最小的不构成回路的边
- 时间复杂度 $O(e \log e)$
- 适合**稀疏图**

### 7.5 最短路径（必考）

#### Dijkstra 算法

- 单源最短路径，**不允许负权边**
- 时间复杂度 $O(n^2)$
- 每轮确定一个顶点的最短距离

#### Floyd 算法

- 所有顶点间最短路径
- 时间复杂度 $O(n^3)$
- 三重循环：`k`（中转点）、`i`（起点）、`j`（终点）

### 7.6 拓扑排序

- **AOV 网**：顶点表示活动，边表示先后关系
- 每次选入度为 0 的顶点输出
- 存在环 → 无法拓扑排序

### 7.7 关键路径

- **AOE 网**：边表示活动，顶点表示事件
- 关键路径 = 最长路径 = 决定工期
- 关键活动：最早开始时间 = 最晚开始时间

## 八、查找

### 8.1 顺序查找

- $ASL_{成功} = \frac{n+1}{2}$
- $ASL_{失败} = n+1$
- $O(n)$

### 8.2 二分查找（必考）

- 前提：**有序表 + 顺序存储**
- $ASL \approx \log_2(n+1) - 1$
- 判定树：每次与 mid 比较，构建折半判定树
- $O(\log n)$

### 8.3 分块查找

- 块间有序，块内无序
- $ASL = L_b + L_w$（索引查找 + 块内查找）

### 8.4 二叉排序树 (BST)

- 左 < 根 < 右
- 中序遍历 → 有序序列
- 查找 $O(\log n)$ ~ $O(n)$（取决于树形态）

> [!warning] BST 的删除
> - 叶结点：直接删
> - 单子树：子树替代
> - 双子树：用直接前驱（左子树最大）或直接后继（右子树最小）替代

### 8.5 平衡二叉树 (AVL)

- 平衡因子 = 左高 - 右高，只能为 -1/0/1
- 最小不平衡子树：距插入点最近的不平衡结点

| 旋转 | 场景 |
|:---|:---|
| LL（右单旋） | 在左子树的左孩子插入 |
| RR（左单旋） | 在右子树的右孩子插入 |
| LR（左右双旋） | 在左子树的右孩子插入 |
| RL（右左双旋） | 在右子树的左孩子插入 |

### 8.6 哈希表（必考）

> [!important] 哈希表核心
> 关键字 → 哈希函数 → 地址

**哈希函数构造**：
- 除留余数法：$H(key) = key \bmod p$，p 取不大于表长的质数

**冲突处理**：

| 方法 | 做法 |
|:---|:---|
| 开放定址法 - 线性探测 | $H_i = (H(key) + i) \bmod m$ |
| 开放定址法 - 平方探测 | $H_i = (H(key) \pm i^2) \bmod m$ |
| 链地址法 | 同义词拉成链表 |

**装填因子** $\alpha = \frac{记录数}{表长}$

> [!tip] ASL 计算（考试重点）
> $$ASL_{成功} = \frac{所有关键字查找次数之和}{关键字个数}$$
> $$ASL_{失败} = \frac{每个地址位查找失败所需比较次数之和}{哈希表长度}$$

## 九、排序

### 9.1 排序算法总览

| 排序算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 |
|:---|:---|:---|:---|:---|:---|
| 直接插入 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✅ 稳定 |
| 折半插入 | $O(n\log n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✅ 稳定 |
| 希尔排序 | — | $O(n^{1.3})$ | $O(n^2)$ | $O(1)$ | ❌ 不稳定 |
| 冒泡排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✅ 稳定 |
| 快速排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n^2)$ | $O(\log n)$ | ❌ 不稳定 |
| 简单选择 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ❌ 不稳定 |
| 堆排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(1)$ | ❌ 不稳定 |
| 归并排序 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(n)$ | ✅ 稳定 |
| 基数排序 | $O(d(n+r))$ | $O(d(n+r))$ | $O(d(n+r))$ | $O(r)$ | ✅ 稳定 |

> [!tip] 口诀速记
> - **稳定**：插冒归基（插入、冒泡、归并、基数）
> - **不稳定**：希快选堆（希尔、快排、选择、堆）
> - **$O(n \log n)$**：快堆归并（快排、堆排、归并）
> - **$O(1)$ 空间**：插希冒选堆（插入、希尔、冒泡、选择、堆排）

### 9.2 插入排序

#### 直接插入排序

- 将待排序记录插入到已排序序列的合适位置
- 最好（正序）$O(n)$，最坏（逆序）$O(n^2)$
- 稳定

#### 希尔排序

- 按增量分组，组内直接插入
- 最后增量为 1
- 不稳定

### 9.3 交换排序

#### 冒泡排序

- 每轮将最大元素"冒泡"到最后
- 加 `flag` 优化：一轮无交换即结束
- 稳定

#### 快速排序（必考手算）

> [!important] 快排思想
> 选枢轴 pivot → 划分（左小右大）→ 递归

- 每次划分确定 pivot 的最终位置
- 最坏（基本有序）$O(n^2)$
- 不稳定

### 9.4 选择排序

#### 简单选择排序

- 每轮选最小元素与当前位置交换
- 无论什么情况都是 $O(n^2)$
- 不稳定（交换可能破坏顺序）

#### 堆排序

- **大根堆**：每个结点 ≥ 左右孩子（用于升序排序）
- **小根堆**：每个结点 ≤ 左右孩子（用于降序排序）

> [!note] 堆排序步骤
> 1. 建堆（从 $\lfloor n/2 \rfloor$ 开始向下调整）
> 2. 堆顶与末尾交换，向下调整
> 3. 重复直到只剩一个元素

- 建堆 $O(n)$，每次调整 $O(\log n)$，总 $O(n \log n)$
- 不稳定
- 空间 $O(1)$

### 9.5 归并排序

- 二路归并：两两合并有序序列
- 分治思想
- 稳定，$O(n \log n)$，但需要 $O(n)$ 辅助空间

### 9.6 基数排序

- 不基于比较，基于关键字各位
- LSD（最低位优先）或 MSD（最高位优先）
- 稳定
- 适合关键字可拆分且范围不大的情况

### 9.7 排序算法选择策略

| 场景 | 推荐算法 |
|:---|:---|
| n 较小 (n ≤ 50) | 直接插入、简单选择 |
| n 大，要求稳定 | 归并排序 |
| n 大，关键字随机 | 快速排序 |
| n 大，关键字基本有序 | 直接插入、冒泡 |
| n 大，只取前 k 个 | 堆排序 |
| 空间有限 | 堆排序 |
| 关键字可拆分 | 基数排序 |

## 十、易错点总结

> [!danger] 易错清单
> - 栈和队列逻辑上是**线性结构**（操作受限）
> - 完全二叉树 ≠ 满二叉树
> - 最小生成树不唯一（权值相同边）
> - 快排**不稳定**（枢轴交换破坏顺序）
> - 堆排序**不稳定**（堆顶交换破坏顺序）
> - 循环队列判空/判满条件不同
> - 图的 DFS 用栈，BFS 用队列
> - 哈夫曼树 WPL 速算：所有**非叶结点**权值之和
> - KMP next 数组手算：最长相等前后缀长度 + 1

---

> [!success] 复习建议
> 1. 先背熟复杂度总表 + 二叉树性质
> 2. 手算题型：哈夫曼树、遍历恢复树、KMP next、哈希表 ASL、快排划分过程
> 3. 代码重点关注：二叉树遍历（递归+非递归）、链表操作、排序算法
> 4. 判断题拉分点：稳定 vs 不稳定、各种"一定"/"可能"的表述
