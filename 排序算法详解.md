# 排序算法详解

## 1. 概述

排序算法按策略分为两类：

| 类型 | 算法 | 时间复杂度（平均） |
|------|------|-------------------|
| **比较排序** | 冒泡、选择、插入 | $O(n^2)$ |
| **比较排序** | 归并、快速、堆排序 | $O(n \log n)$ |
| **非比较排序** | 计数、基数、桶排序 | $O(n + k)$ |

- **稳定性**：相等元素排序后相对位置不变 → 稳定排序
- **原地排序**：$O(1)$ 额外空间 → 原地排序

---

## 2. $O(n^2)$ 基础排序

### 2.1 冒泡排序（Bubble Sort）

**思路**：相邻两两比较，每次遍历把最大元素"冒泡"到末尾。

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:  # 提前终止优化
            break
```

| 指标 | 值 |
|------|-----|
| 最优时间 | $O(n)$（已有序，一趟无交换） |
| 最差时间 | $O(n^2)$ |
| 空间 | $O(1)$ |
| **稳定** | ✅ 是（相邻相等不交换） |

### 2.2 选择排序（Selection Sort）

**思路**：每轮找到未排序部分的最小值，放到已排序末尾。

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
```

| 指标 | 值 |
|------|-----|
| 最优/最差 | 均 $O(n^2)$ |
| 空间 | $O(1)$ |
| **稳定** | ❌ 否（交换可能跨越相等元素） |

> **对比冒泡**：选择排序交换次数少（每轮最多1次），但无法提前终止。

### 2.3 插入排序（Insertion Sort）

**思路**：类似扑克牌理牌，将当前元素插入前面已排序的合适位置。

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
```

| 指标 | 值 |
|------|-----|
| 最优时间 | $O(n)$（基本有序） |
| 最差时间 | $O(n^2)$ |
| 空间 | $O(1)$ |
| **稳定** | ✅ 是 |

> **最佳场景**：小规模数据、基本有序的数据。也是[希尔排序](希尔排序.md)和高级排序（快排小数组）的基础。

## 3. $O(n \log n)$ 高级排序

### 3.1 归并排序（Merge Sort）

**思路**：分治——递归拆分数组 → 合并两个有序子数组。

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return merge(left, right)


def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

| 指标 | 值 |
|------|-----|
| 最优/最差 | 均 $O(n \log n)$ |
| 空间 | $O(n)$（临时数组） |
| **稳定** | ✅ 是（`<=` 保证） |

> **适用**：链表排序（不需要额外空间）、外部排序、需要稳定性的场景。

### 3.2 快速排序（Quick Sort）

**思路**：选 pivot → partition（小的左，大的右）→ 递归左右子数组。

```python
def quick_sort(arr, low=0, high=None):
    if high is None:
        high = len(arr) - 1
    if low < high:
        pi = partition(arr, low, high)
        quick_sort(arr, low, pi - 1)
        quick_sort(arr, pi + 1, high)


def partition(arr, low, high):
    pivot = arr[high]  # 选最后一个为基准
    i = low - 1        # i 指向小于 pivot 的区域末尾
    for j in range(low, high):
        if arr[j] < pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1
```

| 指标 | 值 |
|------|-----|
| 最优时间 | $O(n \log n)$ |
| 最差时间 | $O(n^2)$（pivot 选到最小/大） |
| 空间 | $O(\log n)$（递归栈） |
| **稳定** | ❌ 否 |

**优化策略**：
- **三数取中**：选 `arr[low]`、`arr[mid]`、`arr[high]` 的中位数作 pivot
- **小数组切插入排序**：`if high - low < 10: insertion_sort(arr[low:high+1])`
- **三路快排**：处理大量重复元素时分为 `小于/等于/大于` 三区

```python
# 三路快排（Dutch National Flag）
def three_way_qsort(arr, low, high):
    if low >= high:
        return
    lt, gt = low, high
    pivot = arr[low]
    i = low
    while i <= gt:
        if arr[i] < pivot:
            arr[lt], arr[i] = arr[i], arr[lt]
            lt += 1; i += 1
        elif arr[i] > pivot:
            arr[gt], arr[i] = arr[i], arr[gt]
            gt -= 1
        else:
            i += 1
    three_way_qsort(arr, low, lt - 1)
    three_way_qsort(arr, gt + 1, high)
```

### 3.3 堆排序（Heap Sort）

**思路**：建最大堆 → 反复取堆顶（最大值）放末尾 → 调整堆。

```python
def heap_sort(arr):
    n = len(arr)
    # 建堆：从最后一个非叶节点开始
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    # 排序
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)


def heapify(arr, n, i):
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)
```

| 指标 | 值 |
|------|-----|
| 最优/最差 | 均 $O(n \log n)$ |
| 空间 | $O(1)$ |
| **稳定** | ❌ 否 |

> **对比快排**：堆排序无最差退化，原地排序，但常数因子更大，缓存不友好。

## 4. 非比较排序（线性时间）

### 4.1 计数排序（Counting Sort）

**条件**：数据范围 $k$ 有限（如 0~100 的成绩）。

```python
def counting_sort(arr):
    if not arr:
        return arr
    max_val = max(arr)
    count = [0] * (max_val + 1)
    for num in arr:
        count[num] += 1
    # 前缀和 → 确定每个元素的位置
    for i in range(1, len(count)):
        count[i] += count[i - 1]
    # 从后往前遍历，保证稳定性
    output = [0] * len(arr)
    for num in reversed(arr):
        count[num] -= 1
        output[count[num]] = num
    return output
```

| 指标 | 值 |
|------|-----|
| 时间 | $O(n + k)$ |
| 空间 | $O(n + k)$ |
| **稳定** | ✅ 是（反向遍历时） |

### 4.2 基数排序（Radix Sort）

**思路**：按位依次排序（LSD：从低位到高位），每次用稳定排序（通常计数排序）。

```python
def radix_sort(arr):
    if not arr:
        return arr
    max_val = max(arr)
    exp = 1
    while max_val // exp > 0:
        counting_sort_by_digit(arr, exp)
        exp *= 10
    return arr


def counting_sort_by_digit(arr, exp):
    n = len(arr)
    count = [0] * 10
    for num in arr:
        digit = (num // exp) % 10
        count[digit] += 1
    for i in range(1, 10):
        count[i] += count[i - 1]
    output = [0] * n
    for num in reversed(arr):
        digit = (num // exp) % 10
        count[digit] -= 1
        output[count[digit]] = num
    for i in range(n):
        arr[i] = output[i]
```

| 指标 | 值 |
|------|-----|
| 时间 | $O(d \cdot (n + k))$，$d$ 位数，$k=10$ |
| 空间 | $O(n + k)$ |
| **稳定** | ✅ 是 |

### 4.3 桶排序（Bucket Sort）

**思路**：数据均匀分布到桶 → 桶内排序 → 合并。

```python
def bucket_sort(arr, bucket_size=10):
    if not arr:
        return arr
    min_val, max_val = min(arr), max(arr)
    bucket_count = (max_val - min_val) // bucket_size + 1
    buckets = [[] for _ in range(bucket_count)]
    for num in arr:
        idx = (num - min_val) // bucket_size
        buckets[idx].append(num)
    result = []
    for bucket in buckets:
        result.extend(sorted(bucket))  # 桶内用插入排序
    return result
```

| 指标 | 值 |
|------|-----|
| 最优时间 | $O(n + k)$ |
| 最差时间 | $O(n^2)$（全落一桶） |
| **稳定** | 取决于桶内排序 |

## 6. 场景速查

| 场景 | 推荐算法 | 理由 |
|------|---------|------|
| 数据量小（$n < 50$） | **插入排序** | 常数为 $O(n^2)$ 中最优 |
| 基本有序 | **插入排序** / 冒泡 | 接近 $O(n)$ |
| 需要稳定 | **归并** / 计数 / 基数 | 稳定排序首选 |
| 内存极度受限 | **堆排序** | $O(1)$ 空间 + 无退化 |
| 通用场景（内存够） | **快速排序** | 实践中最快 |
| 外部排序（磁盘） | **归并排序** | 顺序IO友好 |
| 整数、范围不大 | **计数排序** | 线性时间 |
| 大范围整数 | **基数排序** | 位拆分降范围 |
| 浮点、均匀分布 | **桶排序** | 期望线性 |
| 大量重复元素 | **三路快排** | 重复越多越快 |
| 求 Top-K | **堆排序**（部分） | 维护大小 K 的堆 |

## 7. 常见面试题

### 7.1 快速排序 vs 归并排序

- 两者都是分治，主要区别：
  - 快排 **先分区再递归**，归并 **先递归再合并**
  - 快排不稳定，归并稳定
  - 快排原地，归并需临时数组
  - 快排最差 $O(n^2)$，归并严格 $O(n\log n)$

### 7.2 链表用哪种排序？

**归并排序** — 链表可以 $O(1)$ split + merge，无额外空间问题。

### 7.3 为什么快排实践中最快？

- **缓存友好**：顺序访问 partition 两端
- **常数小**：无额外分配 + 精巧循环
- **平均表现**：partition 的随机性让退化极罕见

### 7.4 堆排序为什么比快排慢？

- 跳跃访问 → 缓存不友好
- heapify 的常数大于 partition
- 但无最差退化 → **嵌入式 / 实时系统首选**
